  <h2 id="scope" class="mb-3 scroll-mt-24 font-display text-3xl text-clay-strong leading-[1.1]">
    Or scope it to a subtree
  </h2>
  <p class="mb-5 max-w-2xl text-clay-default text-sm leading-relaxed">
    Three idioms, each one sharper than the last. <strong>Default</strong> wraps in a
    <code class="rounded border border-clay-hairline bg-clay-base px-1 py-px font-mono text-[0.8125rem] text-clay-default">display: contents</code> div (no layout box).
    <strong>asChild</strong> merges the scope onto an existing child via Radix Slot
    (no extra DOM at all).
    <strong>Manual</strong> spreads the raw <code class="rounded border border-clay-hairline bg-clay-base px-1 py-px font-mono text-[0.8125rem] text-clay-default">themeToCssVars(...)</code> object onto any element you control.
  </p>

  <ScopeTabs />

  <p class="mt-6 max-w-2xl text-clay-default text-sm leading-relaxed">
    All three return a <em>complete</em> CSS-variable map (registry defaults + theme
    overrides), so a globally-applied theme can never bleed into the scope through
    a token your theme didn't override. This is why the gallery above can show 16 cards
    side-by-side, each isolated.
  </p>
</section>

  </h2>
  <p class="mb-5 max-w-2xl text-clay-default text-sm leading-relaxed">
    Every preset is plain JSON. The minimum is the four identity fields; every section
    below them is optional, so a colour-only theme stays small while a showcase theme
    can retune geometry, borders, motion, focus, and per-component tokens.
  </p>
  <CodeSource
    code={`interface ThemeConfig {
  id: string;
  name: string;
  description: string;
  accentSwatches: readonly string[];

  // All sections optional.
  colors?: {
    light?: Readonly<Record<string, string>>;
    dark?:  Readonly<Record<string, string>>;
  };
  geometry?: { radius?, spacing?, fontSans?, fontMono?, … };
  borders?:  { width?, style? };
  motion?:   { duration?, easing? };
  focus?:    { width?, offset? };

  components?: Partial<Record<ComponentName, Record<string, string>>>;
}`}

  <h2 id="persist" class="mb-3 scroll-mt-24 font-display text-3xl text-clay-strong leading-[1.1]">
    Remembering the choice
  </h2>
  <p class="max-w-2xl text-clay-default text-sm leading-relaxed">
    Clay stores the active theme id under the <code class="rounded border border-clay-hairline bg-clay-base px-1 py-px font-mono text-[0.8125rem] text-clay-default">clay-theme</code> key in <code class="rounded border border-clay-hairline bg-clay-base px-1 py-px font-mono text-[0.8125rem] text-clay-default">localStorage</code>. The picker reads it on mount and re-applies the selection. If storage is unavailable, the theme falls back to <code class="rounded border border-clay-hairline bg-clay-base px-1 py-px font-mono text-[0.8125rem] text-clay-default">default</code> for the session.
  </p>
